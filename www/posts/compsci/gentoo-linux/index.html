Gentoo Linux

Wed Jan 04, 2023 · 467 words

Gentoo Linux is a source distribution, in other words, everything is compiled from the source code.

Overview ¶

Many of us are used to running .exe files, but where do they come from? If you try to open a binary with a text editor, all you’ll get is a string of rubbish interpreted by your computer. So first thing we do is write a code in a programming “language”, which is compiled into the machine language. In a source distribution, every binary has to be compiled from the source code.

Installation ¶

Like many distributions, you are spared from compiling the base linux, which would be the case for Linux from Scratch. You start with a ready system and all you need to do is to make it bootable. This is, inter alia, installing a bootloader (if you don’t have one yet) and compiling the kernel. Then you need to configure it, more or less.

  1. Acquire the base, or in Gentoo’s terms, a stage 3.

You can find them on the main gentoo page. Each one of them has a profile, with systemd/openrc, multilib/no multilib, selecting a profile now saves time for later.

  1. Configure Portage. Set up the repository.

It’s possible to have many, but our first and default shall be official Gentoo repository. Portage, the heart of Gentoo, keeps information about packages in forms of files. From that, it maintains both dependencies and features that can be compiled in to a binary. It’s a brilliant tool for managing packages, something you will start missing when installing Linux from Scratch.

  1. Update the base system.

Once we have the flags tuned based on our computer, update the system. Optionally, get a text editor of your liking, since nano is the default.

  1. Configure your system.

Take your time setting up the: timezone, locales, networking.

  1. Compile the kernel and the bootloader.

When you install the kernel, the only thing that Portage is going to do is download the source code for you. The configuration is done through ncurses, a very neat terminal-based interface. Now, the bootloader can be any program you want, even the Windows bootloader will work. To let your system know what partitions to mount where, /etc/fstab needs finetuning.

Minus points ¶

The things that could be considered as cons: compilation times, steep learning curve, time-consuming activity, criticisms like “you have way too much free time to be compiling all this”. As a matter of fact, compiling chromium engine took me 7.5 hours on a 4-core laptop.

Let’s not get carried away, realistically, you won’t be compiling packages 24/7.

This is more or less how the installation looks like, this is pretty much what the video is. Should you use it as a daily driver? I’d recommend it to people that want perfect command of their non-redundancy toolset, to perfectionists definitely.
