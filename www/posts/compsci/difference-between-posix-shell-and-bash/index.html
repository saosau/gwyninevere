Difference Between POSIX Shell and Bash

Tue Dec 20, 2022 · 224 words

When writing a script, all that is told to people is to write #!/bin/bash on the first line. What’s worse is when people start to think that #!/bin/sh and #!/bin/bash are interchangeable.

POSIX compliance ¶

So, I like to think of POSIX as a set of rules. You can be POSIX compliant in two ways: Bash is POSIX compliant, because it can run any script that complies to POSIX rules, and a script is POSIX compliant if it only has capabilities specified in POSIX rules.

Arrays or double bracket test statements [[ condition ]] aren’t POSIX. To check for illegal statements you can use shellcheck .

What is #!/bin/sh ¶

So, the /bin/sh, it’s not really a shell, it’s a link to any shell of your choice. So in your computer it might be Bash, on the other Dash, who knows.

#! is what we call a shebang. When you call an executable like this: ./executable, the exec function looks for the first 2 bytes: #!. If it hits it, it runs your text file with a program that comes after it.

Thus, if you are writing a bash script, it can only be ran by Bash, so start your first line with #!/bin/bash. If you are trying to make your shell script work on other POSIX compliant shells, then you can start it with #!/bin/sh.
