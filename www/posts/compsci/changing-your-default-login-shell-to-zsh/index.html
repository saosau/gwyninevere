Changing Your Default Login Shell to Zsh

Sat Oct 22, 2022 · 671 words

Since we went over choosing a POSIX shell, now it is time we looked into interactive shells that greet us every morning. The three very often mentioned shells are Bash, Zsh and Fish. I adapted to Zsh fairly quickly, mainly because it was POSIX compliant, zshrc is basically a copy of my bashrc. Fish is an entirely different shell that needs learning new syntax. I am not fond of it, but it should not stop you in any way.

Benefits of using Zsh to Bash ¶

Autocompletion. Pretty syntax highlighting. If you are in for that, you should check it out, as Bash tab autocompletion is just autofilling and printing out glob really. Zsh will provide you with a selection menu that – yes – can be controlled by hjkl. With syntax highlighting you will know whether the command exists and check for typos.

I do not think it is worth to compare speeds, because user experience is the most important in interactive shells. Nevertheless, Zsh runs pretty quick anyway.

Change your default shell to Zsh ¶

This is a very weird one. I keep all my startup environment definitions, e.g. PS1, LSCOLORS in a ~/.profile. So it baffled me that Zsh ignores it and only looks for ~/.zprofile. Because I want to be able to switch between shells easily, I do not hardcode .profiles. Either way, my login is done by /bin/sh, in this case Dash. Then in .profile if you set SHELL to Zsh, every terminal spawned will be Zsh. Lastly in the end I run exec, to make existing process Zsh. If you do not like it, the “normal” way is to use chsh.

# Last updated: 25/08/2022
[ "${-#*"i"}" = "$-" ] && return # non-interactive shell
export PATH=$HOME/Build/dwm/bin:$PATH
export SHELL='/bin/zsh' # If I want to switch shells, I do it here.

The bashrc translated into zshrc ¶

Here are both of my RCs:

Differences more in detail ¶

  • HISTFILE can be anything else, if not specified it will just create a default one.
  • fast-syntax-highlighting is a plugin that you can get on github. It comes with many colorschemes.
  • In Bash, we put various codes or, as you see, you can use tput between \[ \]. On the other hand, in Zsh the format is %{ %}. Both of these yield exactly the same PS1. To use fg you need to load color module.
  • Lines that follow are for autocompletion.
  • Keybindings are pretty much the same, just invoked differently with different parameters.
