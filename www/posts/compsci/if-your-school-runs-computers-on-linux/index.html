If Your School Runs Computers on Linux

Wed Dec 14, 2022 · 426 words

Once I had some time to spare, I decided to spend it on hacking the computers in our school laboratories. While happiness would take over me after I succesfully connected my laptop to eduroam via wpa_supplicant , I began to consider installing dwm without sudo.

Creating a local repository ¶

The school computers run Debian, so it was hella long since I have used apt. I would use these commands most frequently:

apt-cache search [package] # search packages
apt-get source [package] # fetch package source

which was pretty nice, because the package versions were compatible with current Debian release. Source packages usually have a configure script, with which you can easily set compile flags just like Gentoo USE flags. To have everything installed in your local repository just add the flag: --prefix=[directory] .

Compiling source code ¶

Honestly, it quickly turned into a mundane work, because of dependencies. dwm depends on X11 libraries, of which I had libXft and libxinerama missing. libXft depends on libxrender, freetype and fontconfig, moreover each one of them required either another library or a build time dependency. Usually pkg-config does the work with CFLAGS/LDFLAGS, but sometimes I had to go so far to edit pkg-config files themselves. You usually: 1) include libraries with -I[directory] , 2) link libraries with -L[directory] . Obviously, I worked this out by writing an lmao.c file that included local libraries.

I had times where I wanted to give up, but in actuality I just went to WC to wash my face and continued my suffering. After some time I managed to get it all going, once I set it up finally dwm appeared. I couldn’t contain it anymore, so I left the room to forget that I have done it. With thus achieved experience I installed feh as well for my dwm background. It was an easy run since it was also written in C, with things ready I found a pretty cool Dark Souls vector wallpaper.
